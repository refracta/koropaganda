# **Koropaganda v0.0.1**
Koropaganda는 한국기술교육대학교의 학교 홍보 문구에서 영감을 받아 만들어진 난해한 프로그래밍 언어입니다.

## **개요**  
---  
가제 Koropaganda(이하 코로파간다)는 한국기술교육대학교의 홍보를 위해 작성된 프로그래밍 언어이다.  
코로파간다는 네 개의 스택과 열 개의 리터럴 그리고 열한 개의 명령줄을 제공한다.  

## **스택**  
---  
제공되는 스택은 다음과 같다.  
* 공기업 스택  
    * 표준 입력 스택으로 사용자의 인수를 불러온다.

* 중소기업 스택  
    * 명령줄 매개변수를 저장하는 스택으로 연산 등에 활용된다.

* 중견기업 스택  
    * 사용자가 데이터를 저장할 수 있는 스택으로 출력 시에도 활용된다.

* 대기업 스택  
    * 리턴 스택으로 연산 결과를 저장하는 스택이다.

## **리터럴**  
---  
### **기본 리터럴**  
기본 리터럴에는 KOREATECH 리터럴과 SKY 리터럴이 포함된다.  
* KOREATECH 리터럴
    * KOREATECH  
    * 한국기술교육대학교  
    * 한기대  

상기된 세 개의 리터럴은 모두 동일한 지위를 가지며 상호 변경하여 사용이 가능하다.  
아래에 작성된 세 구문은 모두 동치이다.
```
한기대가 한기대했다!
한기대가 한국기술교육대학교했다!
KOREATECH이 KOREATECH했다!
```

* SKY 리터럴
    * SKY  

---
### **취업 리터럴**  
취업 리터럴은 사용된 명령어에 따라서 다른 의미를 갖는다.  
연산과 관련된 구문에서는 정수 리터럴로서 사용되고 저장 및 출력과 관련된 구문에서는 스택 포인터로서 사용된다.  

* 정수 리터럴로서의 취업 리터럴
    * 공기업: 0  
    * 중소기업: 1  
    * 중견기업: 2  
    * 대기업: 3  

* 스택 포인터로서의 취업 리터럴
    * 공기업: Standard Input Stack  
    * 중소기업: Parameter Stack  
    * 중견기업: Data Stack  
    * 대기업: Return Stack  

---
### **대학 리터럴**  
대학 리터럴은 연산 시 부호를 결정하는 리터럴이다.
* 대학: +  
* 대학원: -  

## **명령어**  
---  
제공되는 명령어 구문은 다음과 같다.  
* {취업}보다 {기본}  
    > 취업 리터럴에 해당하는 스택의 가장 마지막 두 요소를 비교한 평가값을 중소기업 스택에 넣는다.  
    > 가장 마지막 요소가 크면 참 작으면 거짓을 반환한다.  
    > 단, 기본 리터럴 요소는 무시된다. 즉 다음의 코드는 서로 동치이다.  
    >  ```
    > 대기업이 한기대했다!
    > 대기업이 KOREATECH했다!
    > ```
    > 
    > 대기업 스택이 [ 1, 2, 4 ]일 때의 사용 예:
    > ```
    > 대기업이 한기대했다!  
    > ```
    > 중소기업 스택이 비었다고 가정했을 때, 위의 코드를 실행하면 중소기업 스택은 다음과 같이 변화한다.  
    >
    >   > [ 1 ]

* {SKY}보다 {KOREATECH}  
    > 입학 홍보처에 의하면 스카이보다 코리아텍이므로 중소기업 스택에 1을 넣는다.  

* {KOREATECH}보다 {SKY}  
    > 입학 홍보처에 의하면 코리아텍보다 스카이는 참은 아니므로 중소기업 스택에 -1을 넣는다.  

* {기본}보다 {기본}  
    > 입학 홍보처에 의하면 정의할 수 없으므로 중소기업 스택에 0을 넣는다.  

* {취업} 취업률 (압도적) Y위 (Z%)  
    > 후술할 공식을 이용해 값을 연산하여 중소기업 스택에 넣는다.  
    > 단, 이때의 취업 리터럴은 정수 리터럴로서 사용된다.
    >   > '압도적' 키워드가 없을 때,  
    >   > {취업} * Y의 Z% 만큼을 중고기업 스택에 넣는다.  
    >  
    >   > '압도적' 키워드가 있을 때,  
    >   > {취업} ^ Y의 Z% 만큼을 중고기업 스택에 넣는다.  
    >
    > Z% 인수를 전달하지 않을 경우 앞선 연산값의 100%를 중소기업 스택에 넣는다.  
    > 단, 소수점은 올림 처리한다.
    > ```
    > 중견기업 취업률 압도적 3위 40%
    > 대기업 취업률 19위
    > ```
    > 위의 식을 평가해보면 각각 다음과 같다.  
    >   > 2^3의 40%인 3.2를 올림하여 4  
    >
    >   > 3 * 19의 100%인 57를 올림하여 57  
    > 
    > 즉 위의 코드가 실행되면 4와 57이 차례대로 중소기업 스택에 들어간다.  

* {대학} 취업률 (압도적) Y위 (Z%)  
    > 후술할 공식을 이용해 값을 연산하여 평가된 개수만큼 중소기업 스택에서 값을 더해 특정 스택에 넣는다.  
    > 압도적 키워드가 있으면 중소기업에 없으면 대기업에 값을 넣는다.  
    > 대학 키워드가 있으면 더한 값의 양의 값을 대학원이 있으면 음의 값을 취한다.  
    > > Y의 Z% 만큼 중소기업 스택에서 더해 다른 스택에 넣는다.  
    > 
    > Z% 인수를 전달하지 않을 경우 앞선 연산값의 100%를 중소기업 스택에 넣는다.  
    > 단, 소수점은 올림 처리한다.  
    > 중소기업 스택이 [ 1, 2, 4 ]일 때의 사용 예:  
    > ```
    > 대학 취업률 2위!
    > 대학원 취업률 압도적 3위!
    > ```
    > 위 식을 평가하면 각각 다음과 같다.
    >   > 2의 100%가 2이므로 중소기업 스택의 마지막 두 요소를 더한다.  
    >   > 대학 키워드를 사용하였으므로 더한 값의 양의 값을 스택에 넣는다.  
    >   > 압도적 키워드가 없으므로 중소기업에 스택에 값을 넣는다.  
    >  
    >   > 3의 100%가 3이므로 중소기업 스택의 마지막 세 요소를 더한다.  
    >   > 대학원 키워드를 사용하였으므로 더한 값의 음의 값을 스택에 넣는다.  
    >   > 압도적 키워드가 있으므로 대기업 스택에 값을 넣는다.  
    > 
    > 중소기업 스택과 대기업 스택의 변화는 다음과 같다.
    >
    >   > 중소기업 스택: [ 1, 2, 4 ]  
    >   > 대기업 스택: [ ]  
    >
    >   > 중소기업 스택: [ 1, 2, 4, 6 ]  
    >   > 대기업 스택: [ ]  
    >
    >   > 중소기업 스택: [ 1, 2, 4, 6 ]  
    >   > 대기업 스택: [ -12 ]  
    >

* {KOREATECH}이 {SKY}했다!
    > 표준 출력으로 중견기업 스택의 가장 마지막에 있는 요소를 유니코드 문자로 출력한다.  
    > 출력된 요소는 스택에서 삭제된다.  
    > 
    > 중견기업 스택이 [ 65 ]일 때의 사용 예:
    > ```
    > 한기대가 SKY했다!
    > ```
    > 표준 출력을 통해 'A'가 출력된다.  

* {SKY}가 {KOREATECH}했다!
    > 표준 에러로 중견기업 스택의 가장 마지막에 있는 요소를 유니코드 문자로 출력한다.  
    > 출력된 요소는 스택에서 삭제된다.  
    > 
    > 중견기업 스택이 [ 97 ]일 때의 사용 예:
    > ```
    > SKY가 한기대했다!
    > ```
    > 표준 에러를 통해 'a'가 출력된다.  

* {KOREATECH}이 {KOREATECH}했다!
    > 정상적인 프로그램 종료.  
    > 종료 코드 0으로 프로그램을 종료한다.  

* {SKY}가 {SKY}했다!
    > 비정상적인 프로그램 종료.  
    > 종료 코드 1으로 프로그램을 종료한다.  

* {취업}이 {기본}했다!
    > 취업 리터럴에 해당되는 스택의 가장 마지막 요소룰 출력 또는 반환없이 삭제한다.  

* {기본}이 {취업}했다!
    > 취업에 해당되는 스택에서 값을 읽어 해당 값만큼 앞선/뒷선 명령줄을 실행한다.  
    > 단 주석 또는 공백은 줄로 생각하지 않는다.  
    >
    > 중소기업 스택이 [ 1, 2, -1 ]일 때의 사용 예:  
    > ```
    >  1 | KOREATECH보다 SKY
    >  2 | SKY보다 KOREATECH
    >  3 | <2021년 교육부 대학 알리미 공시>
    >  4 |
    >  5 |
    >  6 | 한기대가 중소기업했다!
    >  7 |
    >  8 |
    >  9 |
    > 10 |
    > ```
    > 6번째 줄에서 구문을 만나면 중소기없 스택의 마지막 요소가 -1이므로 다음 명령은 7번째 줄이 아닌 이전 명령인 2번째 줄의 명령이 실행된다.  

* {취업}이 {취업}했다!
    > 첫 번째 인자에 해당되는 스택의 가장 마지막 요소를 pop하여 두 번째 인자에 해당되는 스택에 push한다.  

* <%주석% 공시>  
    > 주석 구문이다.  
    > 주석 구문은 프로그램 실행에 아무런 영향을 주지 않으며 무시된다.
